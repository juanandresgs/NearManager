# Near v0.2.0

Near v0.2.0 is the first public-source release candidate of the backend-independent keyboard-first TUI platform and its Far-inspired reference file manager.

## Supported release artifacts

- macOS 13 or later on Apple Silicon (`macos-aarch64`).
- macOS 13 or later on Intel (`macos-x86_64`).
- Current stable Ubuntu-compatible Linux on x86-64 (`linux-x86_64`).
- Current stable Windows on x86-64 with ConPTY support (`windows-x86_64`).

Source builds remain available for other Rust-supported targets, but a successful build does not imply operator qualification.

## Included applications

- `near-fm`: dual-panel Far-inspired file manager and platform integration reference.
- `near-view`: standalone streaming text and hexadecimal viewer.
- `near-proc`: non-filesystem process-provider reference application.
- `near-demo`: backend-independent custom-surface and interaction proof.

## User-visible changes

- Resumable, revision-bound qualification with explicit platform degradation and operator-evidence records.
- Multiple retained embedded-terminal tabs, pane placement, focus switching, and reversible `Ctrl+O` zoom.
- Durable ten-slot Temporary Panels integrated with side-targeted `Alt+F1` and `Alt+F2` location workflows; clearing a slot removes references and never source files.
- Shared collection lookup now prefers filename prefixes and falls back to visibly identified internal matches.
- Native macOS Trash restoration, mounted-root mutation denial, and clearer capability routing.
- Public installation, contribution, security, licensing, release verification, SBOM, provenance, and attestation paths.

## Configuration and plugin migration

The v0.1 configuration and plugin protocol versions remain accepted. No automatic migration is required. Back up user configuration before testing a pre-release build.

## Compatibility breaks

- Terminal workspace commands and screen labels reflect retained tabs rather than a singleton user screen.
- Release archive names identify macOS architecture explicitly.
- APIs not listed as published in `docs/near-api-compatibility.md` remain internal and may change without a compatibility shim.

## Known limitations

- Near remains pre-release. Production qualification is exact-revision and platform-specific; this release note does not replace generated qualification evidence.
- Linux and Windows behavior is constrained by native metadata, Trash or Recycle Bin, shell, and terminal capabilities.
- Arbitrary split trees, terminal-session restart restoration, pane mouse forwarding, and a cross-agent message bus are not yet implemented.
- Process-extension sandbox enforcement is strongest on macOS; unsupported platforms fail closed where equivalent isolation is unavailable.

## Security and verification

Each archive includes the two Near license texts and top-level README. Release assets also include SHA-256 checksums, portable provenance, dependency and license inventories, an SPDX SBOM, and a GitHub artifact/SBOM attestation. See `docs/INSTALL.md` for verification commands.

## Rollback

Restore the previous application binaries and configuration-directory backup. Near performs registered configuration migrations atomically and retains the last valid configuration when reload fails.
